## 3. Project Proposal

**Format:** 1–2 page PDF.

Proposal may include:

- **Background:** Application domain or research area. Why is this problem important? Specific questions you aim to address.
- **Data:** Dataset you will use. Key characteristics — features, labels, number of samples, etc. Is it publicly available, and how will you access it?
- **Proposed Work:** Main focus of the project. Will evaluation be based primarily on implementation of existing methods, development of new ideas, or extensive experimentation? Clearly describe expected contributions.
- **Team Members Name**

**Possible dataset sources:** Public repositories (Kaggle, UCI, etc.).

**Submission:** One team member submits the proposal on Canvas.

### Mandatory Requirement

- Students **must consult with the instructor within the first week** to finalize and approve the dataset.

## 4. Final Report

**Format:** 5 pages max, excluding bibliography.

Report may include:

- **Introduction:** Background of the project and the current state of knowledge in the area. Include the application domain (healthcare, finance, computer vision, etc.) and why the problem is important.
- **Problem Definitions:** Clearly define the ML problem. Include: classification / regression / clustering / other ML task, input features and target variable (if applicable), the goal of the model, and any assumptions or constraints.
- **Proposed Methods:** Methodology used. Include: overview of the ML pipeline, models used, why these methods were chosen, any feature engineering or transformation techniques, and hyperparameter tuning or validation strategies.
- **Data Understanding & Preprocessing:** How the dataset was worked with. Include: data cleaning steps (missing values, duplicates, outliers), feature engineering or selection, data visualization insights, and train-test split strategy.
- **Model Development:** How the models were built. Include: training process, algorithms implemented, baseline vs. improved model, any optimization techniques used, and justification of model choices.
- **Model Evaluation:** Evaluate using appropriate metrics and compare models.
- **Results:** Summarize findings, implications, and limitations of the approach.
- **Conclusion:** Concise summary of the entire project with key takeaways.
- **Team Contributions:** Brief summary of how each team member contributed.

### Submission Requirements

Submit a ZIP file on Canvas including:

1. Final report (PDF format)
2. Source code or a link to a GitHub repository
3. Dataset or dataset link

## 5. Final Project Presentation

**Length:** 10 minutes total.

- **8 minutes** — Presentation
- **2 minutes** — Q&A session

Presentation should include:

- **Introduction and Problem Statement:** Briefly explain the project topic, application area, and the problem being addressed.
- **Dataset Description:** Describe the dataset used, including features, target variable, and source of the data.
- **Methodology:** Summarize the ML techniques and models applied, along with key preprocessing steps.
- **Model Development:** How the models were trained; important design choices or improvements.
- **Results:** Main evaluation metrics and comparison of model performance.
- **Conclusion:** Most important findings and what they mean in the context of the problem.

The presentation should be clear, well-structured, and supported by visuals such as slides, charts, or diagrams.
